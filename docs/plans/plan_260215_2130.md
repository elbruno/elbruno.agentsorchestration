# Plan: Canvas-Based Agent Chart with 3-Tab Layout

**Date**: 2026-02-15 21:30  
**Status**: In Progress

## TL;DR

Replace the SVG-based `AgentGraph.razor` with an HTML5 Canvas rendering engine (JS interop). All 11 agents always visible in a two-ring layout (Orchestrator center, 6 core inner ring, 5 specialists outer ring). Animated bezier lines draw between agents on activation; active nodes glow/pulse. Restructure the right panel into 3 tabs: Agent Chart, Implementation Plan, Call Flow.

## Steps

### Phase 1: Canvas JS Module

1. **Create `samples/AspireApp/Web/wwwroot/js/agent-canvas.js`** — self-contained JS module exposing `window.agentCanvas` with:
   - `init(canvasId, dotNetRef)` — setup 2D context, calculate two-ring positions, start `requestAnimationFrame` loop, register click hit-testing
   - `setNodes(nodes[])` — receive 11 agent definitions, compute positions (Orchestrator center, 6 core inner ring r≈140px, 5 specialists outer ring r≈240px)
   - `activateNode(name, color)` — glow animation (pulsing border, color wash)
   - `completeNode(name)` — green border, stop pulse
   - `resetNode(name)` — return to idle
   - `drawConnection(from, to, color, animated)` — animate bezier line via stroke-dashoffset; active connections pulse with traveling particle
   - `deactivateConnection(from, to)` — fade to dim gray
   - `deactivateAll()` — reset all
   - `resize()` — recalculate on container resize via ResizeObserver
   - `dispose()` — cleanup

2. **Register script** in `App.razor` — add `<script src="js/agent-canvas.js"></script>`

### Phase 2: Blazor Component

1. **Create `samples/AspireApp/Web/Components/AgentCanvasChart.razor`** — Blazor wrapper:
   - Renders `<canvas>` filling container
   - `OnAfterRenderAsync` → `agentCanvas.init()` + `agentCanvas.setNodes()`
   - Parameters: `Nodes`, `Connections`, `OnNodeClicked`
   - `[JSInvokable] OnCanvasNodeClicked(string name)`
   - Exposes `ActivateNodeAsync()`, `CompleteNodeAsync()`, `DrawConnectionAsync()` etc.
   - Implements `IAsyncDisposable`

2. **Extend `DashboardModels.cs`** — add `AnimationState` to `AgentConnection` (*parallel with step 3*)

### Phase 3: 3-Tab Layout

1. **Restructure `Home.razor` tabs** to 3 tabs: (*depends on step 3*)
   - "Agent Chart" (default) → `<AgentCanvasChart>` replacing `<AgentGraph>`
   - "Implementation Plan" → `<PlanViewer>` + `<PhaseProgress>` (moved from overlay)
   - "Call Flow" → `<AgentFlow>` (unchanged)
   - Update `ActiveTab` values: `"chart"`, `"plan"`, `"flow"`

2. **Remove `<AgentGraph>` reference** from Home.razor. Keep the component file for backward compat. (*parallel with step 5*)

### Phase 4: Wire Events to Canvas

1. **Update `ApplyEvent()` in Home.razor** (*depends on step 5*):
   - `AgentActivatedEvent` → `ActivateNodeAsync()` + `DrawConnectionAsync()`
   - `AgentCompletedEvent` → `CompleteNodeAsync()` + `DeactivateConnectionAsync()`
   - `AgentCommunicationEvent` → `DrawConnectionAsync(from, to)`
   - `OrchestrationCompletedEvent` → `DeactivateAllAsync()`

### Phase 5: Styling & All-Agents

1. **Add CSS** in `app.css` for canvas container + tab active states (*parallel with step 7*)

2. **Update `AgentStyles` dictionary** in Home.razor to include all 11 roles (*parallel with step 7*)

3. **Update `ResetStatuses()`** to initialize all 11 agents at startup (*depends on step 9*)

## Decisions

- HTML5 Canvas + JS interop (not SVG) for richer animation capabilities
- Two-ring layout: center + inner (6 core) + outer (5 specialist)
- PlanViewer moves to dedicated tab (no longer overlaid)
- Old AgentGraph.razor kept but unreferenced
- No external charting library — self-contained JS module
- Excluded: drag-to-rearrange, zoom/pan, force-directed physics
