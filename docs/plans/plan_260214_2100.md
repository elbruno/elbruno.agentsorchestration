# Plan: Full Development Cycle — Fixer Agent, AppRunner, Aspire & ServiceDefaults

**Date:** 2026-02-14 21:00  
**Status:** Completed

## Summary

Extend the orchestration dashboard to support a complete application development cycle: **chat → plan → code → build → fix → launch → iterate**. Adds a Fixer agent with configurable retry loop, an AppRunner for launching generated apps, Aspire AppHost for service orchestration, ServiceDefaults (based on ZavaServiceDefaults), and a Minimal API project.

## Phase 1: Fixer Agent & Retry Loop (Core)

### 1.1 Add Fixer AgentRole

- Add `Fixer = 4` to `AgentRole` enum
- Create `Instructions/fixer.md` — role: analyze build errors, produce corrected code
- Update `InstructionLoader.LoadFromDirectory()` to include Fixer
- Update `AgentConfigurationStore.CreateDefaultInstructions()` and `CreateDefaultConfigurations()`
- Add Fixer case to `TemplateAgentClient` switch

### 1.2 Fix .csproj Template Bug

- `TemplateAgentClient.BuildProjectFile()` produces invalid XML `<Project Sdk="Microsoft.NET.Sdk">.Web`
- Fix to use proper `Sdk="Microsoft.NET.Sdk.Web"` attribute

### 1.3 Add New Orchestration Events

- `FixAttemptStartedEvent(DateTimeOffset, int Attempt, string BuildOutput)` — emitted when fix loop begins
- `FixAttemptCompletedEvent(DateTimeOffset, int Attempt, bool Success, string Output)` — emitted after each fix attempt

### 1.4 Retry Loop in OrchestrationService

- Add `MaxFixAttempts` configuration (default: 3, from `Orchestration:MaxFixAttempts` appsettings)
- After build fails in `VerifyAsync`, enter fix loop:
  1. Emit `FixAttemptStartedEvent`
  2. Activate Fixer agent with build error log
  3. Fixer produces corrected files
  4. Write corrected files to workspace
  5. Re-run `dotnet build`
  6. Emit `FixAttemptCompletedEvent`
  7. If still failing and attempts < max, repeat
- Final build result (pass/fail) is used for the orchestrator summary

### 1.5 AppRunner (Workspace/AppRunner.cs)

- `LaunchAsync(workspacePath)` — starts `dotnet run` as a background process
- `StopAsync()` — kills the process
- `IsRunning` property
- `AppLaunchedEvent`, `AppStoppedEvent`, `AppLogEvent` — streamed to UI
- Timeout and port configuration from appsettings
