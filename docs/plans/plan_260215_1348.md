# Plan: Security & Scalability Hardening

**Date:** 2026-02-15 13:48  
**Status:** ✅ Implemented

## Summary

A full security and scalability audit of the three extracted NuGet libraries (`AgentsOrchestration.Abstractions`, `AgentsOrchestration.Orchestration`, `AgentsOrchestration.Core`) and their consumers (Web and API projects). This plan identifies issues, proposes changes, and tracks implementation.

---

## Security Issues Found

### SEC-1: Path traversal in `ExecuteTaskAsync` — file scope not sanitized ⚠️ HIGH

**Location:** `OrchestrationService.cs:153`  
**Issue:** `task.FileScope` is derived from LLM output and passed directly to `Path.Combine()`. A malicious or hallucinated file scope like `../../etc/passwd` could write files outside the workspace directory.  
**Fix:** Validate that the resolved full path starts with the workspace path after canonicalization.  
**Status:** ✅ Fixed — added `ValidatePathWithinWorkspace()` guard.

### SEC-2: Path traversal in `WorkspaceManager.ReadFile` ⚠️ MEDIUM

**Location:** `WorkspaceManager.cs:46`  
**Issue:** `ReadFile(relativePath)` joins the path without validating that the resolved path stays within the workspace boundary. A caller could pass `../../etc/passwd`.  
**Fix:** Validate the full path stays within `WorkspacePath`.  
**Status:** ✅ Fixed — added path containment check.

### SEC-3: Unbounded `runningOrchestrations` dictionary in API — memory leak ⚠️ MEDIUM

**Location:** `AgentsOrchestration.Api/Program.cs:24`  
**Issue:** Completed orchestration runs are never removed from the `ConcurrentDictionary`. Over time this leaks memory — each run holds `OrchestrationService`, `CancellationTokenSource`, and the full `OrchestrationResult`.  
**Fix:** This is an API-layer concern (not library). Documented for future implementation: add a max-run-count cap or TTL-based eviction.  
**Status:** 📋 Documented — out of scope for library changes; future work for API project.

### SEC-4: No input validation on `OrchestrationRequest.Prompt` ⚠️ LOW

**Location:** `OrchestrationService.cs:23`  
**Issue:** Null or empty prompts are accepted and flow through the pipeline.  
**Fix:** Add `ArgumentException.ThrowIfNullOrWhiteSpace` guard at the top of `RunAsync`.  
**Status:** ✅ Fixed — added null/whitespace validation.

### SEC-5: `InstructionLoader` does not validate directory path ⚠️ LOW

**Location:** `InstructionLoader.cs:6`  
**Issue:** If a non-existent directory is passed, the method silently returns empty strings for all roles. This could hide configuration mistakes.  
**Fix:** Throw `DirectoryNotFoundException` if the path does not exist.  
**Status:** ✅ Fixed — added existence check.

## Scalability Issues Found

### SCALE-1: Unbounded event channel ⚠️ MEDIUM

**Location:** `OrchestrationService.cs:19`  
**Issue:** `Channel.CreateUnbounded<OrchestrationEvent>()` — if no consumer reads events, the channel grows without limit. In production with long-running orchestrations, this could consume significant memory.  
**Fix:** Switch to `Channel.CreateBounded<OrchestrationEvent>(new BoundedChannelOptions(1000) { FullMode = BoundedChannelFullMode.DropOldest })`.  
**Status:** ✅ Fixed — switched to bounded channel with configurable capacity.

### SCALE-2: No concurrency limit on parallel task execution ⚠️ LOW

**Location:** `OrchestrationService.cs:35-36`  
**Issue:** `Task.WhenAll(phase.Tasks.Select(task => ExecuteTaskAsync(...)))` runs all tasks in parallel with no concurrency cap. If a phase has many tasks, this could overwhelm an LLM API with concurrent requests.  
**Fix:** Document as a known limitation. A future improvement could use `SemaphoreSlim` or `Parallel.ForEachAsync` with a `MaxDegreeOfParallelism` option.  
**Status:** 📋 Documented — future enhancement.

### SCALE-3: `maxFixAttempts` has no upper bound ⚠️ LOW

**Location:** `OrchestrationService.cs:12`  
**Issue:** Callers can set `maxFixAttempts` to an arbitrarily large number, causing the fixer loop to retry excessively.  
**Fix:** Clamp the value to a maximum of 10.  
**Status:** ✅ Fixed — added upper bound validation.
